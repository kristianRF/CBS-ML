Our goal is to develop a data-driven model which can segment customers from the last summers data and use it to recommend events to this summers new bookings.

Naive Bayes¶

Naive Bayes is supervised learning algorithm based on applying Bayes' theorem with the naive assumption of conditional independence between every pair of variables/features given the class in the response variable.

Bayes' theorem revisit¶

Let $A$ and $B$ be two events or variables. Then the conditional probability of $A$ given $B$ can be computed by $$\mathbb{P}(A | B) = \frac {\mathbb{P}(A, B)}{\mathbb{P}(B)}= \frac {\mathbb{P}(B | A) \times \mathbb{P}(A)}{\mathbb{P}(B)}.$$ In many statistics reference, the above probabilities can be interpreted as follows: - $\mathbb{P}(A|B)$ Posterior: How probable is our hypothesis given the observed evidence? - $\mathbb{P}(B|A)$ Likelihood: How probable is the evidence given that our hypothesis is true? - $\mathbb{P}(A)$ Prior: How probable is the hypothesis before observing evidence? - $\mathbb{P}(B)$ Marginal: How probable is the new evidence under all possible hypotheses?

Probabilistic classification with maximum a posterior¶

Suppose there are total $K$ classes in the response variable $y$ and there are $D$ features/predictive variables. The maximum a posterior (MAP) classification rule can be simply expressed as follows: $$\mathbb{P}(y = c_k \mid \boldsymbol{X} ) \geq \mathbb{P}(y = c_{-k} \mid \boldsymbol{X}),$$ where $c_k$ represents class $k$ and $c_{-k}$ represents any class expect class $k$.
\begin{align*} \mathbb{P}(y=c_ | \boldsymbol{X}) & \ = \frac{\mathbb{P}(\boldsymbol{X} | y=c_) \mathbb{P}(y=c_)}{\mathbb{P}(\boldsymbol{X})}\\ & \ \propto \mathbb{P}(\boldsymbol{X} | y=c_) \mathbb{P}(y=c_) \\ & \ \propto \prod_{d=1}^{D} \mathbb{P}(x_d | y=c_) \mathbb{P}(y=c_) && \triangleright~\text{Conditional independence}\\ \end{align*}

Specification of prior¶

The prior $\mathbb{P}(y=c_)$ can be specified directly from the response variable in the training set as follows:

\begin{align*} \mathbb{P}(y=c_) & \ = \frac{\sum_{n=1}^{N} \mathbb{I}(y_n = c_)}{N}, \end{align*}

where $\mathbb{I}(\cdot)$ is the indicator function, that is

$$ I(\cdot) = \begin{cases} 1, \text{if } \cdot \text{ is true},\\ 0, \text{otherwise}. \end{cases}$$

Specification of likehood¶

The specification of the likehood $\mathbb{P}(x_d | y=c_)$ is different for categorical and numerical variables.

Gaussian Naive Bayes¶

If variable $X^{(d)}$ is a numerical variable, the likelihood $\mathbb{P}(X^{(d)} = x \mid Y=c_)$ is assumed to be Gaussian: $$\mathbb{P}(X^{(d)} = x \mid Y=c_) = \frac{1}{\sqrt{2\pi\sigma^2_{d}}} \exp\left\{-\frac{(x - \mu_{d})^2}{2\sigma^2_{d}}\right\},$$ where $\mu_{d}$ and $\sigma^2_{d}$ are the mean and variance of variable $X^{(d)}$, which can be obtained through maximum likelihood estimation.

We can write the likelihood function for variable $X^{(d)}$ as follows

\begin{align*} \mathscr{L}(\mu_{d}, \sigma^2_{d}; X^{(d)}_1 = x^{(d)}_1, \cdots, X^{(d)}_N = x^{(d)}_N) & = \prod_{n=1}^{N} \frac{1}{\sqrt{2\pi\sigma^2_{d}}} \exp\left\{-\frac{(x_n - \mu_{d})^2}{2\sigma^2_{d}}\right\} \\ & = (2\pi\sigma^2_{d})^{-\frac{N}{2}} \prod_{n=1}^{N} \exp\left\{-\frac{(x_n - \mu_{d})^2}{2\sigma^2_{d}}\right\} \\ \end{align*}

Therefore, the log-likelihood function is then

\begin{align*} LL(\mu_{d}, \sigma^2_{d}; X^{(d)}_1 = x^{(d)}_1, \cdots, X^{(d)}_N = x^{(d)}_N) & = -\frac{N}{2} \ln\{2 \pi\} -\frac{N}{2} \ln \{\sigma^2_d\} -\frac{1}{2\sigma^2_{d}} \sum_{n=1}^{N} (x_n - \mu_{d})^2.\\ \end{align*}

Then the maximum likelihood estimators of the mean and the variance are \begin{align*} \hat{\mu}_{d} & = \frac{1}{N} \sum_{n=1}^{N} x_n,\\ \hat{\sigma}^2_{d} & = \frac{1}{N} \sum_{n=1}^{N} (x_n - \hat{\mu}_{d})^2.\\ \end{align*}

Categorical Naive Bayes¶

Suppose there are $T$ categories in variable/feature $x^{(d)}$. The probability of category $t$ in variable/feature $x^{(d)}$ given class $c_k$ in the response varaible can be estimated as follows: $$\mathbb{P}(x^{(d)} = t \mid Y=c_) = \frac{ \sum_{n=1}^{N}\mathbb{I}(x^{(d)}_n = t, y_n =c_k) + \alpha}{\sum_{n=1}^{N}\mathbb{I}(y_n = c_k) + \alpha T},$$ where $\alpha$ is the called the smoothing parameter. If $\alpha = 1$, it is called Laplacian smoothing.

CategoricalNB implements the categorical naive Bayes algorithm for categorically distributed data. It assumes that each feature, which is described by the index , has its own categorical distribution.

For each feature in the training set , CategoricalNB estimates a categorical distribution for each feature i of X conditioned on the class y. The index set of the samples is defined as , with as the number of samples.

K-nearest neighbors¶

K-nearest neighbors (in short KNN) falls in the supervised learning family of algorithms. It is a non-parametric and instance-based learning algorithm.

Non-parametric means it makes no explicit assumptions about the functional form, avoiding the dangers of mismodeling the underlying distribution of the data. For example, suppose our data is highly non-Gaussian but the learning model we choose assumes a Gaussian form. In that case, our algorithm would make extremely poor predictions.

There are many ways to define model type. An important question is: does the model have a fixed number of parameters or does the number of parameters grow with the amount of training data? The former is called a parametric model, and the latter is called a non-parametric model. Parametric models have the advantage of often being faster to use, but the disadvantage of making stronger assumptions about the nature of the data distributions. Non-parametric models are more flexible, but often computationally intractable for large datasets. Kevin Murphy. Machine Learning: A Probabilistic Perspective. MIT Press, 2014, Chapter 1

Instance-based learning (also called memory-based learning) means that our algorithm does not explicitly learn a model. Instead, it chooses to memorize the training instances which are subsequently used as knowledge for the prediction phase. Concretely, this means that only when a query to our database is made, will the algorithm use the training instances to spit out an answer.

Mathematical formulation¶

Given an input $\boldsymbol{x}$ from the test set
    1:   Choose criteria $K$
    2:   Compute the distances from the input $\boldsymbol{x}$ to the labeled examples in the training set.
    3:   Sort the distances of labeled examples in an increasing order.
    4:   Calculate the probabilities of the top $K$ nearest neighbors classes: $$ \mathbb{P}(y = c | \boldsymbol{x}, \mathscr{D}, K) = \frac{1}{K} \sum_{n \in \mathscr{N}_K(\boldsymbol{x}, \mathscr{D})} \mathbb{I}{(y_n = c)}, $$          where $\mathscr{N}_K(\boldsymbol{x}, \mathscr{D})$ is the set of the top $K$ nearest points to $\boldsymbol{x}$ in $\mathscr{D}$ and $\mathbb{I}{(\cdot)}$ is the indicator function.
    5:   Provide the estimate of the class.

Metrics for distance computation¶

  • Eucildean Distance (or $L^2$-norm) $$ || \boldsymbol{x} - \boldsymbol{x}_n ||_2 = \sqrt{ \sum_{d = 1}^D (\boldsymbol{x}^{(d)} - \boldsymbol{x}^{(d)}_n)^2} $$

  • Mahantan Distance $$ || \boldsymbol{x} - \boldsymbol{x}_n ||_1 = \sum_{d = 1}^D \big|\boldsymbol{x}^{(d)} - \boldsymbol{x}^{(d)}_n \big| $$

  • Minkowski Distance (or $L^p$-norm) $$ || \boldsymbol{x} - \boldsymbol{x}_n ||_p = \bigg[\sum_{d = 1}^D \Big|\boldsymbol{x}^{(d)} - \boldsymbol{x}^{(d)}_n \Big|^q \bigg]^{1/q} $$
